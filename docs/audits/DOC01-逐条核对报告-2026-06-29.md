## 18. 修复优先级建议

P0：必须先补，否则 DOC-01 不能算商用闭环完整。

1. 补 Cloudflare `_redirects`：`/forgot-password`、`/reset-password`。
2. 登录态移除协议勾选阻断，只保留账号、密码、记住我、返回首页。
3. 注册字段把“名字”调整为“账号名称 *”，补固定说明、右侧检测中/可用/不可用状态。
4. 邮箱和手机号增加前端实时查重。
5. 验证码增加尝试次数，超过 5 次失效。
6. 协议和隐私政策改成真实链接，并预留 `/terms`、`/privacy` 页面。

P1：提升到文档里的商用细节。

1. 行业词库从代码内小列表升级为数据库 500+ 词库。
2. 临时邮箱黑名单扩展到 200+。
3. 敏感词库扩展到 200+。
4. 弱密码库扩展到 500+ 以上。

7. 重置密码页补“链接失效 -> 重新发送”完整状态。

P2：工程层面的商用升级。

1. DOC-01 页面从单 HTML 过渡实现迁移到 React 18 + TypeScript + Tailwind 的独立页面。
2. 用 E2E 浏览器测试覆盖注册、登录、退出、找回、重置、移动端响应式。
3. 线上接入真实 Resend、Redis、HTTPS、CORS 后做一次完整生产验收。
